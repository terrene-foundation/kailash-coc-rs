# Trust-Plane Cross-Language Binding Patterns

## Binding Parity Principle

All binding users are equal. Every feature available in Python MUST be available in Ruby, Node.js, and C ABI. This is a first principle established in Round 2 human decisions.

## Thread Safety by Binding

### Python (PyO3)

- **GIL release for file I/O**: Long-running file operations release the GIL via `py.allow_threads()`
- This allows other Python threads to run during trust project I/O

```rust
// Pattern in kailash-python
fn decide(&self, py: Python, action: &str) -> PyResult<PyVerdict> {
    py.allow_threads(|| {
        self.inner.decide(action, &context)
    }).map(|v| PyVerdict(v))
    .map_err(|e| PyErr::new::<PyRuntimeError, _>(e.to_string()))
}
```

### Ruby (Magnus)

- **Clone-before-compare for `is_tighter_than`**: Prevents deadlock when comparing constraint envelopes
- Ruby's GVL (Global VM Lock) means a single OS thread runs Ruby code at a time
- If `is_tighter_than` held a lock on self while accessing other, deadlock could occur

```rust
// Pattern in kailash-ruby — clone BEFORE comparison
fn is_tighter_than(&self, other: &RbConstraintEnvelope) -> bool {
    let self_clone = self.inner.clone();
    let other_clone = other.inner.clone();
    self_clone.is_tighter_than(&other_clone)
}
```

### Node.js (napi-rs)

- **u32 cast for AtomicUsize**: napi-rs represents numbers as f64 or u32, not usize
- Lifetime counters (AtomicUsize) are cast to u32 in the binding
- This is documented as safe because lifetime_evaluations < u32::MAX in practice

```rust
// Pattern in kailash-node
#[napi]
pub fn shadow_records_limited(&self, count: u32) -> Vec<JsShadowRecord> {
    let records = self.inner.shadow_records();
    records.iter()
        .rev()
        .take(count as usize)
        .map(|r| JsShadowRecord::from(r))
        .collect()
}
```

### C ABI (kailash-capi)

- **Panic-safe**: All 21 C ABI functions use `std::panic::catch_unwind`
- **Null-on-panic**: Out pointers are set to null if a panic occurs inside the function
- **JSON data exchange**: All complex types are serialized to JSON strings
- **Opaque pointers**: `TrustProject` is exposed as `*mut c_void`

```c
// C header (generated by cbindgen)
int tp_shadow_enable(TrustProjectHandle handle, const char* config_json);
int tp_shadow_report(TrustProjectHandle handle, char** out_report);
int tp_shadow_disable(TrustProjectHandle handle, char** out_report);
int tp_shadow_check(TrustProjectHandle handle, const char* action, const char* ctx_json, int* out_verdict);
int tp_shadow_promote(TrustProjectHandle handle);
```

## Shadow Mode in Bindings

All bindings expose project-level shadow methods:

1. `enable_shadow(config)` — Enable shadow mode with candidate config
2. `disable_shadow()` — Disable shadow mode
3. `shadow_report()` — Get divergence report
4. `shadow_check(action, context)` — Shadow-aware action check
5. `shadow_promote()` — Promote candidate to production
6. `shadow_records()` — Get all shadow records
7. `is_shadow_active()` — Check if shadow mode is active

Node.js additionally has `shadow_records_limited(count)` for pagination of large record sets.

## Threshold Configuration in Bindings

All bindings expose `ShadowConfig` with configurable thresholds:

```python
# Python example
from kailash.trust_plane import ShadowConfig

config = ShadowConfig(
    max_records=10000,
    persist_records=True,
    min_samples_for_recommend=100,
    promote_threshold=0.05,
    revert_threshold=0.20,
)
project.enable_shadow(candidate_envelope, config)
```

## Key Gotchas

1. Python: Always release GIL for file I/O — `py.allow_threads()`
2. Ruby: Always clone before comparison — never hold locks across method calls
3. Node.js: AtomicUsize → u32 cast is safe in practice but loses data above u32::MAX
4. C ABI: Always check return code AND null-check out pointers
5. WASM binding has minimal trust-plane support (3 types) — use native bindings for full functionality
